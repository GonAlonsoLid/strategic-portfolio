## Directory Purposes

**config/**
- Purpose: Single source of truth for all parameters (paths, model hyperparameters, backtest settings, feature windows)
- Contains: `config.yaml` only; YAML format for human readability
- Key files: `config/config.yaml`

**data/raw/**
- Purpose: Input datasets (not generated by code, manually placed)
- Contains: Excel file with S&P 500 events (joiners/leavers 1994–2026), large CSV with daily CRSP-style price data
- Key files: `data/raw/SPX_index leavers & joiners_17-Feb-2026.xlsx`, `data/raw/daily.csv`

**data/interim/**
- Purpose: Intermediate outputs during pipeline execution
- Contains: Daily panel with returns, market cap, volume, and index membership state
- Key files: `data/interim/daily_panel.parquet` (cached to avoid recomputation)

**data/processed/**
- Purpose: Final feature and model output datasets
- Contains: Feature matrices with labels (join/leave), model probability scores
- Key files: `features_join.parquet`, `features_leave.parquet`, `join_scores.parquet`, `leave_scores.parquet`

**results/figures/**
- Purpose: Visual outputs (PNG plots for research reports)
- Contains: Event study CAR plots (pre/post announcement, inclusion reversal), backtest cumulative returns and drawdown curves
- Key files: `event_study_car_*.png`, `cumulative_returns.png`, `drawdown.png`

**results/tables/**
- Purpose: Tabular outputs (CSV tables for research reports and presentations)
- Contains: CAR by relative day, backtest summary metrics (return, Sharpe, max DD, VaR, skewness), subperiod analysis, factor loadings
- Key files: `backtest_summary_predictive.csv`, `backtest_subperiod_3y.csv`

**src/data/**
- Purpose: Data ingestion and transformation
- Contains: Raw data loaders (Excel events, CSV prices with chunking), daily panel builder with S&P 500 membership tracking
- Key files: `src/data/load_data.py` (loaders), `src/data/preprocess_data.py` (panel builder)

**src/events/**
- Purpose: Event study analysis (abnormal returns, CAR)
- Contains: Event window generator, AR/CAR computation, aggregation by event type and relative day, plotting functions
- Key files: `src/events/event_study.py` (main pipeline), `src/events/event_windows.py` (window generator)

**src/features/**
- Purpose: Feature engineering and label creation
- Contains: Rolling momentum/volatility/liquidity, quality proxy, market cap rank, forward-looking join/leave labels
- Key files: `src/features/feature_engineering.py` (main), `src/features/rolling_features.py` (individual feature functions)

**src/models/**
- Purpose: Binary classification (join/leave prediction)
- Contains: Model instantiation (logistic, RF, GB, XGBoost), rolling time-series splits, metric computation
- Key files: `src/models/join_prediction.py`, `src/models/leave_prediction.py`, `src/models/model_utils.py` (splits/metrics)

**src/portfolio/**
- Purpose: Portfolio construction from model predictions
- Contains: Long/short weight builder, weighting schemes (equal, probability, risk-parity, rank), rebalance scheduling
- Key files: `src/portfolio/portfolio_construction.py` (main), `src/portfolio/weighting_schemes.py` (weight functions)

**src/backtesting/**
- Purpose: Portfolio simulation with realistic costs
- Contains: Daily return calculation from weights, turnover computation, transaction cost deduction
- Key files: `src/backtesting/backtester.py` (Backtester class), `src/backtesting/transaction_costs.py` (cost function)

**src/evaluation/**
- Purpose: Performance analysis and attribution
- Contains: Metric compilers (Sharpe, max DD, VaR, skewness, subperiod analysis), optional Fama-French factor regression
- Key files: `src/evaluation/performance_metrics.py` (main metrics), `src/evaluation/factor_analysis.py` (factor attribution)

**src/utils/**
- Purpose: Shared utilities across modules
- Contains: YAML config loading, nested dict traversal, plotting utilities
- Key files: `src/utils/config_loader.py` (config management), `src/utils/plotting.py` (CAR, returns, drawdown plots)

**scripts/**
- Purpose: Executable entry points for pipeline stages
- Contains: `run_event_study.py` (standalone), `train_models.py` (standalone), `run_backtest.py` (full pipeline)
- Key files: All three are independent; `run_backtest.py` is the comprehensive orchestrator

**notebooks/**
- Purpose: Interactive research and exploration
- Contains: Event study CAR visualization, feature distribution analysis
- Key files: `notebooks/01_event_study.ipynb`, `notebooks/02_feature_exploration.ipynb`

**docs/**
- Purpose: Research documentation and methodology
- Contains: Research design notes, Hedge Funds syllabus mapping, extension suggestions
- Key files: `docs/RESEARCH_NOTES.md`

## Key File Locations

**Entry Points:**
- `scripts/run_backtest.py`: Full pipeline orchestrator; check cache, execute stages sequentially
- `scripts/run_event_study.py`: Standalone event study (load panel/events, compute CAR, plot)
- `scripts/train_models.py`: Standalone model training (load features, rolling splits, save scores)

**Configuration:**
- `config/config.yaml`: All parameters (paths, event windows [pre=60, post=60], feature lookbacks [21,63,126,252 days], model types, backtest params)

**Core Logic:**

*Data Layer:*
- `src/data/load_data.py`: load_events() reads Excel and normalizes to [event_type, ticker, event_date]; load_prices_chunked() iterates CSV
- `src/data/preprocess_data.py`: build_daily_panel() tracks S&P 500 membership (state machine: initial constituents + ADD/DEL events)

*Feature Layer:*
- `src/features/feature_engineering.py`: build_feature_panel() adds all rolling features, rank, labels; build_joiner_label() and build_leaver_label() look 63 trading days ahead
- `src/features/rolling_features.py`: add_momentum_features(), add_volatility_features(), add_quality_proxy(), etc.

*Model Layer:*
- `src/models/join_prediction.py`: run_join_prediction() trains multiple classifiers with rolling splits, saves scores
- `src/models/model_utils.py`: make_rolling_splits() (5 years train, 1 year test, no overlap), train_and_evaluate() (fit, predict, compute ROC-AUC/precision/recall/F1)

*Portfolio Layer:*
- `src/portfolio/portfolio_construction.py`: build_long_short_portfolio() creates target weights (long top 10% joiners, short top 10% leavers)
- `src/portfolio/weighting_schemes.py`: equal_weight(), probability_weight(), risk_parity_weight(), rank_weight() functions

*Backtest Layer:*
- `src/backtesting/backtester.py`: Backtester class with run_backtest() method; forward-fills rebalance weights to daily, computes daily return and turnover
- `src/backtesting/transaction_costs.py`: estimate_costs() deducts bps × turnover from returns

*Evaluation Layer:*
- `src/evaluation/performance_metrics.py`: compute_performance_metrics() (annualized return, Sharpe, max DD, VaR, skewness), compute_subperiod_metrics()
- `src/evaluation/factor_analysis.py`: run_factor_regression() (optional Fama-French attribution)

**Testing:**
- `notebooks/` used for exploratory validation; no unit tests (research project)

## Naming Conventions

**Files:**
- `load_data.py`, `preprocess_data.py`: Verb-noun pattern for data pipeline stages
- `feature_engineering.py`, `rolling_features.py`: Domain-specific names (engineering = builder, rolling = implementer)
- `join_prediction.py`, `leave_prediction.py`: Explicit target names (what is being predicted)
- `portfolio_construction.py`, `weighting_schemes.py`: Construction = builder, schemes = variants
- `backtester.py`: Agent noun (the thing that backtests)
- `transaction_costs.py`: Noun phrase (what is being computed)
- `performance_metrics.py`, `factor_analysis.py`: Analysis domain names
- `config_loader.py`: Verb-noun pattern (loading config)
- `plotting.py`: Verb-noun pattern (plotting results)

**Directories:**
- `data/{raw,interim,processed}`: Data pipeline stages (raw input → intermediate computation → final output)
- `src/{data,events,features,models,portfolio,backtesting,evaluation,utils}`: Functional domains (clear separation of concerns)
- `scripts/`: Executable entry points (all scripts are run-*.py convention)
- `results/{figures,tables}`: Output types
- `notebooks/`: Interactive exploration (01_*, 02_* numbering for sequence)

**Functions:**
- `load_*()`: Load data from file (load_events, load_prices_chunked, load_config, load_factors)
- `build_*()`: Construct derived data (build_daily_panel, build_feature_panel, build_long_short_portfolio)
- `compute_*()`: Calculate metrics (compute_abnormal_returns, compute_car, compute_performance_metrics)
- `run_*()`: Execute pipeline stage (run_event_study, run_join_prediction, run_backtest)
- `add_*()`: Add feature column (add_momentum_features, add_volatility_features, add_quality_proxy)

**Variables:**
- `cfg` or `config`: Configuration dict (always passed as `config=cfg` parameter)
- `df` or `panel`: DataFrame (data.frame convention from R, used interchangeably)
- `features_join`, `features_leave`: Explicit target-specific feature datasets
- `join_scores`, `leave_scores`: Model output (probability predictions)
- `target_weights`: Portfolio allocation (target allocation, forward-filled to daily)
- `ret` or `returns`: Return time series
- `permno`, `ticker`, `date`: Standard column names in all datasets
- `label_join`, `label_leave`: Binary forward label (1=event occurs in next 63 days, 0 otherwise)

**Constants:**
- `EVENTS_HEADER_ROW = 3`: Excel sheet structure (hardcoded in load_data.py)
- `EVENTS_COL_MAP`: Column name mapping (for Excel flexibility)
- `min_start_year = 2000`: Model training cutoff (in rolling splits)

## Where to Add New Code

**New Feature:**
- Primary code: Add function to `src/features/rolling_features.py` (e.g., `add_custom_feature()`)
- Call from: `src/features/feature_engineering.py` in `build_feature_panel()` before label construction
- Include in: Numeric feature selection in `feat_cols` list (lines 92-100)
- Tests: Validate in `notebooks/02_feature_exploration.ipynb` (distributions, NaN counts)

**New Model:**
- Implementation: Add model class instantiation to `src/models/model_utils.py` (create `_get_model()` entry)
- Copy for join/leave: Update both `src/models/join_prediction.py` and `src/models/leave_prediction.py` to use new model
- Config: Add hyperparameter section to `config/config.yaml` under `models:` (e.g., `custom_model: {param1: value1}`)
- Training: Both prediction modules already iterate over `config["models"]["types"]` list; add to list and model will be trained automatically

**New Weighting Scheme:**
- Implementation: Add function to `src/portfolio/weighting_schemes.py` (signature: `custom_weight(probs: pd.Series, top_decile: float, gross_target: float) -> dict`)
- Integration: Update `src/portfolio/portfolio_construction.py` lines 57-70 to add elif branch for new scheme
- Config: Update `backtest: weighting:` in `config/config.yaml` to include new option

**New Performance Metric:**
- Implementation: Add computation function to `src/evaluation/performance_metrics.py` in `compute_performance_metrics()` dict
- Integration: Metric is automatically included in output CSV (results_tables/backtest_summary_predictive.csv)
- Plotting: If visual, add to `src/utils/plotting.py` and call from `scripts/run_backtest.py`

**New Dataset/Experiment:**
- Caching: Follow pattern in `scripts/run_backtest.py` (check path exists → skip if cached, else compute and save)
- Output format: Use parquet first (faster, typed); fallback to CSV via `try/except ImportError` (see feature_engineering.py line 123-125)
- Intermediate storage: Save to `data/interim/` for temporary outputs; `data/processed/` for final model/feature inputs

**Utilities:**
- Shared helpers: `src/utils/` (config loading, plotting)
- Model helpers: `src/models/model_utils.py` (rolling splits, scaling, metrics)
- Feature helpers: `src/features/rolling_features.py` (individual features)
